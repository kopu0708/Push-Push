## 🥊 PUSH-PUSH (2D 물리 밀어내기 서바이벌)
### "30초의 제한 시간, 차지 대시로 상대를 원판 밖으로 날려버리세요!"
### 유니티 2D 물리 엔진과 코루틴을 활용하여 짜릿한 반동과 타격감을 구현한 탑뷰 아케이드 액션 게임입니다.

## 🎮 게임 룰 (Game Rules)
### 제한 시간: 30초 동안 치열한 밀어내기 공방전이 펼쳐집니다.
 
### 승리 조건: 스페이스바로 기를 모아 대시하여 상대를 원판(Arena) 밖으로 더 많이 떨어트린 쪽이 승리합니다.

### 장외 리스폰: 링 밖으로 떨어지면 상대방이 점수를 획득하며, 떨어진 캐릭터는 즉시 시작 위치로 리스폰되어 전투를 재개합니다.

## 🛠️ 기술 스택 (Tech Stack)
Engine: Unity (2D)

Language: C#

Network: Unity Netcode for GameObjects (NGO), Unity Transport

Version Control: Git / GitHub

## 💡 핵심 구현 기능 (Key Features)
차지 대시 메커니즘: 물리적 관성을 고려한 rb.linearVelocity.magnitude < 0.1f 조건으로 무분별한 대시를 방지하고 전략적 타이밍을 설계했습니다.

직관적인 UI 연동: World Space Canvas와 Fill Amount를 활용하여 실시간 충전 상태를 시각화했습니다.

상태식 AI 및 무적 시스템: 코루틴을 활용해 연산을 최적화하고, 리스폰 시 1초 무적 및 깜빡임 효과를 구현했습니다.

방어적 프로그래밍: MissingReferenceException 방지를 위해 돌진 직전 타겟 상태를 재검증하는 안전장치를 구축했습니다.

동적 UI 구조: 씬과 게임 모드(싱글/로컬)에 따라 텍스트가 유동적으로 변하는 구조를 설계했습니다.

## 🧠 회고 및 배운 점 (Retrospective)
-- 배운 점
물리/성능 최적화: 물리 레이어 매트릭스를 활용한 정석 무적 판정법과 코루틴을 통한 연산 최적화의 중요성을 배웠습니다.

엔진 라이프사이클 이해: Time.timeScale이 글로벌 환경에 미치는 영향과 씬 전환 시 필요한 상태 초기화 규칙을 체득했습니다.

UI/씬 시스템: 씬 호출 시의 빌드 인덱스 관리 및 UI 스크립트와의 연결 방식을 완벽히 이해하게 되었습니다.

-- 아쉬운 점과 향후 과제
네트워크 구현의 난관: Netcode for GameObjects를 활용한 랜선 멀티 구현 과정에서, 데이터 동기화와 네트워크 상태 처리에 어려움을 겪었습니다. 비록 이번 프로젝트에서 완벽한 5인 멀티플레이 구현에는 도달하지 못했지만, 멀티플레이 게임을 구성하는 로비(Lobby)와 전송(Transport) 시스템의 기초를 학습한 것은 큰 수확입니다.

AI 의존성: 네트워크 동기화의 복잡함으로 인해 상당 부분 AI 로직에 의존해야 했던 점이 아쉬움으로 남습니다. 추후에는 오늘 배운 네트워크 접속 환경 세팅 지식을 바탕으로, 더 깊이 있게 동기화 로직을 보완해 나갈 계획입니다.
